Problem Statement

Bagaimana pembatalan booking mempengaruhi perbedaan antara potensi revenue dan revenue aktual, serta segmen mana yang paling berkontribusi terhadap lost revenue?

Key Insights
  • Lead time >120 hari → cancel rate 52% (Risiko Utama)
  • Booking Group (61%) dan Transient (41%) paling berisiko
  • ~50% potensi revenue hilang secara struktural
  • Special Request berkorelasi negatif (-0.24) dengan cancel
Rekomendasi Bisnis
  • Non-refundable policy untuk early booking (>120 hari)
  • Dynamic pricing — naikkan harga saat hunian >70%
  • Segmentasi channel — ketat untuk OTA/Transient
  • Kontrol lead time panjang dengan deposit progresif
  • Long Stay Package — diskon ≥4 malam

ML Pipeline
  • Drop leakage columns (reservation_status, bookingID)
  • Split 80:20 — Train: 66.451 | Test: 16.613
  • One-Hot Encoding fitur kategorikal
  • Standard Scaling — samakan skala fitur numerik
  • Train 3 models: LR, Random Forest, XGBoost
Feature Engineering
  • total_guests = adults + children + babies
  • total_stay = weekend + week nights
  • has_deposit — indikator komitmen pembayaran
  • is_loyal — tamu repeated guest
  • high_cancel_history — riwayat cancel sebelumnya
Model Terpilih: XGBoost

Dipilih karena Recall tertinggi (81.09%) — efektif mendeteksi pembatalan yang sebenarnya. Penting untuk meminimalkan missed cancellation.

Top Feature Importance
  • lead_time (28%) — booking jauh = risiko tinggi
  • deposit_type (18%) — no deposit = mudah batal
  • adr (14%) — harga pengaruhi komitmen tamu
  • prev_cancellations (12%) — riwayat = karakter pelanggan

Tentang SKORIA

Platform AI berbasis Hybrid System untuk menganalisis kesiapan UTBK secara personal — memadukan skor TPS, kondisi psikologis & kebiasaan belajar menjadi strategi dan rencana belajar 8 minggu yang personal.

ML Pipeline (LightGBM)
  • 2.000 data simulasi — 9 fitur base (psiko + belajar)
  • Feature Engineering: 9 → 16 fitur (agregat + interaksi)
  • Target: Indeks Kesiapan Non-Akademik (0–100)
  • Hyperparameter tuning dengan Optuna (100 trials)
  • Anti-overfitting: early stopping + 5-fold CV
Feature Interaksi Kunci
  • Efektivitas_Belajar = Skor_Belajar × Skor_Psiko (28%)
  • Konsistensi = Jam × Hari belajar
  • Stabilitas_Mental = Fokus × Percaya_Diri
  • Kualitas_Latihan = Latihan × Review Soal
Arsitektur Hybrid System
🤖 ML — LightGBM
  • → Input: 16 fitur dari form survey
  • → Output: Indeks Kesiapan = 67.4
  • → Pola non-linear antar fitur
